在时代背景与政策扶植之下，年轻的中国 IT
企业与从业者正视差距，积极学习国外先进经验。与此同时，针对软件开发技术的书籍、杂志、网站等多种形式的媒体也开始发展起来，为软件技术的传播造就了平台。

### 边学边干，中国 IT 企业快马加鞭

快速崛起的中国 IT 行业在各方面都处于追赶者的位置，不管产品、技术还是管理，需要向欧美企业学习的太多。

以 ERP 为例，在整个 1990 年代，是 SAP、Oracle 等外国企业在中国市场引领了 ERP 的潮流。SAP 于 1994 年进入中国，与国内
200 余家企业合作，赢得了联想、康佳、海尔、小天鹅、浙江电力等一批率先实施 ERP 的企业客户1。

为了快速拓展和占领市场，这些国际企业多采用合作分销的模式打造行业生态，例如 Oracle 在 2000 年时全国的合作伙伴包括 6 家增值分销商、72
家独立软件开发商、13 家独立软件销售商、3 家应用软件合作伙伴、180 家授权分销商和 4 家授权培训中心，他们共同构成了基于 Oracle
技术和产品的全国性市场开拓、系统集成、增值开发与技术服务体系2。

这些本土企业很多都有自己的长远目标，与国际产品大厂的合作是他们建设团队、培养能力、进入市场的一步。并且产品厂商为了保障服务质量，也会帮助合作伙伴建设能力，甚至要求合作伙伴派出专职人员与自己的团队一同工作，掌握售前、交付、运维等关键环节的能力。

众多国内企业从分销商、独立软件开发商起步，快速积累了领域、技术、管理能力，培养了一大批企业软件人才，一些企业还逐渐积累出了自己的行业产品。

**同时，国内软件厂商也在抓住机遇快速发展。**

![](https://images.gitbook.cn/40873d10-2535-11e9-baa0-0532c9c91503)

例如联想因应电子商务的浪潮，在企业信息化建设、电子商务应用、电子商务服务等领域不断推出适用的产品和方案3。

用友连续推出基于 Web 和 Java 技术的财务管理软件、ERP 乃至企业信息化服务全面解决方案，并在 2001 年成为首家在 A
股上市的私营软件企业4。

尤其在政府信息化领域，“18号文”明确要求：国家投资的重大工程和重点应用系统，应优先由国内企业承担，在同等性能价格比条件下应优先采用国产软件系统。这就给国内厂商创造了难得的发展机遇。

例如清华同方作为公安信息化“金盾工程”总体设计组成员参与了工程整体设计与实施，提供了包括全方位网络系统解决方案、公安视频监控解决方案和信息化、网络化公安业务应用系统的公安系统综合解决方案5。

中软国际与天津泰达经济开发区合作开发了包含政府服务网上门户、开发区信息交换平台、开发区政企应用平台、开发区电子商务平台的数字化经济开发区解决方案
E-TEDA6。

这些企业抓住了发展机遇，后来都成长为行业的中流砥柱，并为行业储备了一大批技术人才。

对于行业里逐渐流行的
J2EE，庄表伟也有所耳闻，这才抓紧假期时间赶快恶补一下。不过买回来这几本书，读起来简直味同嚼蜡，庄表伟读了几十页就觉得眼皮发沉，索性放下书，上网转悠一下。

打开 CSDN 网站，庄表伟注意到一本书的广告，叫《深度探索 C++ 对象模型》。最近他发现，CSDN

仔细看了一下推荐这本《深度探索 C++ 对象模型》的文章，庄表伟打算改天把这本书也买回来看看。虽说现在的工作用不上 C++，但看见“见识 C++

此时在北京，刘江也在细读这篇推荐文章。刘江是电力出版社的编辑，他对于技术图书枯燥乏味的工具书形象也早有微词，尤其是跟国外一些优秀的计算机图书对比，总觉得冷冰冰的，缺乏吸引力，一堆书放在一起看不出优劣。

### 专业技术媒体初成型

回望 2000 年以前，计算机编程类的专业图书跟其他理工科书籍一样，是陈列在新华书店的一个特定角落里、面向特定的一个小读者群的。

因为完全被视为功能性的工具书，这些书的可读性通常不好，编辑和读者也不太重视作译者（尤其是译者）的名气和品牌。把计算机编程的专业图书和专业作译者打造成精品，这种策划与营销的方式应该说是从华中科技大学出版社和侯捷开始的。

2001 年，侯捷在华中科技大学出版社连续出版了《深入浅出 MFC（第二版）》《深度探索 C++ 对象模型》《Effective
C++》《Essential C++》等著译作，在业内广受关注。尤其是《深入浅出 MFC》，累计销售码洋超过 700 万元人民币（约 9
万册），被评为全国优秀畅销书以及“湖北省 2002 年最有影响的 10 本书”之一7。

《深入浅出 MFC》与《深度探索 C++
对象模型》这两本书，都是对底层技术的抽丝剥茧，内容艰深，且实用性并不强。这两本书的畅销，折射出的是快速成长的软件开发者群体对知识的渴求。

![](https://images.gitbook.cn/08a99790-2534-11e9-8fe8-13e19f1a4ac0)

2000 年时，全国软件行业从业人数仅 18 万人，仅仅 8 年后这个数字已经增长了 10 倍8。

快速成长的产业需要大量的人才，大量新进入这个行业的从业者对知识的需求空前高涨，催生了一批相关的专业网站和媒体， **CSDN**
（中国软件开发者网络）是其中的翘楚。

1999 年，蒋涛和曾登高二人为了解决资料整理、查找麻烦的问题，出版了一套集合了技术文档、文章、源代码、免费软件工具的《程序员大本营》光盘。

在那个几乎所有人都用 56K Modem 拨号上网的年代，这样一套工具光盘很受程序员用户喜爱。而 CSDN
网站起初只是为光盘销售服务的客服网站，后来逐渐发展成了中文世界最大的 IT 技术社区。

2000 年 11 月，CSDN 又推出了 **《程序员》** 杂志，填补了中国 IT 技术综合刊物市场的空白。到 2007 年，CSDN 注册会员已突破
30 万人9。

另一个引人注目的网上社区是 **IBM developerWorks** 网站。

成立于 1999 年的 developerWorks 有一支隶属于市场部的独立编辑团队，除了发布与 IBM
技术和产品相关的资源，还大量发表关于开放技术、尤其是关于 Java 和 Linux 的中立文章10。由于 IBM
的市场经费支持，developerWorks 给国内作者开出的稿费数倍于其他行业媒体，因此积累了一批高质量的内容资源。

一些专注特定技术领域的电子杂志也在这个时期发展起来。

电子杂志可以说是 2000
年前后这一特定历史时期的产物：这个时期软件开发者大多能上网，并且日益重视从互联网上获取信息；同时这个时期宽带尚未普及，很多人还在通过电话拨号上网，因此能离线阅读的电子杂志很受读者青睐。

2001 年，由潘加宇个人建设和维护的 UMLChina 网站发布了重点关注软件需求和设计的电子杂志 **《非程序员》** ，后来共计发行了 51 期，到
2005 年才停刊11。

同样在 2001 年，尚在清华就学的王曦发布了关注 C++ 基础和面向对象设计的电子杂志 **《C++ View》**
，目标是效仿当时国际上颇具影响力的技术杂志 _C++ Report_ 和 _C++ Users Journal_ 12。虽然《C++ View》仅发行 7
期即告终止，但在其发行的这段时间里恰好与侯捷写译的一系列 C++ 深度技术图书所掀起的“C++ 技术热”形成共鸣，传播的范围也很广泛。

### 结语

站在很多年以后回看世纪之交，中国的入世、申奥，与美国的“911”，可能共同构成了一个历史与国运的拐点。身在其时的 IT
人，已经亲身感触到时代的召唤。互联网、政府信息化、企业信息化三股浪潮合力，开启了一个连续十年增长超过 50% 的中国软件黄金年代。

国际大企业在概念、产品、技术等方面都有先发优势，国内厂商通过加盟分销的形式向国际大厂学习，快速积累自己的能力。对软件技术人才的渴求，拉动了传播分享技术知识的出版物和在线社区的发展。十几万像庄表伟和张克强这样的软件从业者，以及像刘江这样对行业保持紧密关注的出版人，对于行业的发展既充满希望、又深感迷茫。高速发展的兴奋与焦虑环绕着每个人，逼着他们奋蹄前行。
